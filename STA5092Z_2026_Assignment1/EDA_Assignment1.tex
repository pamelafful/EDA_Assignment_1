\paragraph{Key Findings:}\label{key-findings}

\textbf{fledg\_survival} dataframe

\begin{itemize}
\item
  Data types in columns are mostly split between character and
  int/numeric data types
\item
  \textbf{Date} column in fledg\_survival is stored as a character- will
  need to be converted. The data format is of the format dd-mm-yy
\item
  \textbf{SurvMassAtInd} column contains null values
\end{itemize}

\textbf{nest\_all} dataframe

\begin{itemize}
\item
  In addition to numeric and int/numeric data types, columns also
  include logical/boolean data types
\item
  \textbf{Date} is also a character data type, similar to the format in
  the \textbf{fledg\_survival} dataframe
\end{itemize}

Merging steps:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
  First, convert date columns in both data frames into datetime columns
  using the lubridate ymd function
\item
  Find common column names in both the \textbf{nest\_all} and
  \textbf{fledg\_survival} data sets to find columns to join on
\item
  Check and ensure that the datatypes of these common columns are the
  same in both data sets- using the janitor package- and change any
  datatype mismatches to the same data type
\item
  Both GrpSizeAD and GrpSizeTotal columns are stored as characters in
  the nest\_all data frame, as they both contain \#NA values
\item
  These will replaced with na values instead, once they are coerced into
  integer values
\end{enumerate}

6) Rate of mass change per day per nesting

Steps for finding these suspicious entries

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  First group Mass by Group and Date to get Mass per Nest over time
\item
  Get lagged values of Date and Mass
\item
  Calculate the percentage Mass change for each row per nest group
\item
  Calculate the date difference, in days, between Date values and lagged
  date values
\item
  Filter for date differences equal to 1 then check which growth rates
  are suspicious (\textgreater50\%)
\end{enumerate}
